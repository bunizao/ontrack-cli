# ontrack-cli v0.1.0

First public release of `ontrack-cli`.

## Highlights

- Terminal-first CLI for Doubtfire / OnTrack deployments
- Browser-session reuse instead of a separate login flow
- First-run `base_url` prompt and validation, aligned with `moodle-cli`
- Automatic browser auth via `refresh_token` exchange when supported
- Project, task, user, auth-check, and teaching-role commands
- JSON and YAML output modes

## Commands

```bash
ontrack user
ontrack auth check
ontrack projects
ontrack project <project_id>
ontrack tasks <project_id>
ontrack roles
```

## Install

```bash
uv tool install ontrack-cli
```

or

```bash
pipx install ontrack-cli
```

## Notes

- Python 3.10+ is required
- Browser auto-auth works best when you are already signed in to the target OnTrack site
- Some deployments may still require explicit `ONTRACK_USERNAME` and `ONTRACK_AUTH_TOKEN`
